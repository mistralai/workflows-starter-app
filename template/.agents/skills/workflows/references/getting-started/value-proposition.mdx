# Mistral Workflows: Value Proposition

Mistral Workflows (SDK + control plane) is our orchestration platform for building production-grade AI workflows. It pairs a durable-execution engine with a developer-friendly Python SDK and native Mistral AI integrations, so you can focus on workflow logic instead of orchestration plumbing.

---

## What Mistral Workflows Gives You

A single platform that combines reliable, durable execution with AI-native capabilities:

- **Simpler DX**: direct activity calls, configuration on decorators, auto-generated schemas
- **Native Mistral AI plugin**: Agents, Tools, Sessions, MCP
- **Real-time streaming**: NATS-backed event/token streaming (token-by-token LLM output, progress updates)
- **AI Studio + Vibe integration**: timeline UI, input forms, callable assistants
- **Interactive primitives**: HITL (Human-in-the-Loop) workflows with `wait_for_input()`
- **Pre-configured observability**: OpenTelemetry traces and dashboards out of the box
- **Auto-registration**: workers register with the control plane on startup
- **Higher-level REST API**: workflows, executions, events, schedules
- **Automatic namespace isolation**: one isolated namespace per org+workspace (`{org_id}:{workspace_id}`)

---

## Durable Execution Foundations

The platform provides battle-tested distributed-systems primitives:

- **Fault Tolerance**: Workflows survive process crashes, network failures, and infrastructure outages
- **Exactly-Once Effects**: Activities execute exactly once despite retries
- **Durability**: Full execution history is persisted; workflows can run for months
- **Deterministic Replay**: Workflow state is reconstructed from event history
- **Automatic Retries**: Configurable retry policies with exponential backoff
- **Signals & Queries**: External interaction with running workflows
- **Child Workflows**: Compose complex workflows from smaller units
- **Schedules**: Cron-like scheduled executions

## Simpler Developer Experience

The SDK removes boilerplate while adding capabilities.

### 1. Direct Activity Calls

Call activities directly — timeout/retry configuration lives on the decorator, not at every call site:

```python
@workflows.activity(retry_policy_max_attempts=3)
async def my_activity(name: str) -> str: ...

# In the workflow:
result = await my_activity(name)  # Just call it
```

### 2. Configuration on Decorators

- Activities: `@activity(start_to_close_timeout=..., retry_policy_max_attempts=..., rate_limit=..., name=...)`
- Workflows: `@workflow.define(name=..., workflow_display_name=..., workflow_description=...)`
- Configure once on the decorator instead of at every call site

### 3. Auto-Generated Schemas

- Auto-generates JSON schemas from function signatures
- Schemas are registered and rendered as AI Studio input forms

### 4. Built-in Dependency Injection

```python
@workflows.activity()
async def my_activity(
    db: Database = Depends(get_database),      # Injected
    client: Mistral = Depends(get_client),     # Injected
) -> Result: ...
```

📖 **More examples:** [Workflows Guide](../guides/workflows) and [Activities Guide](../guides/activities)

## Complete Feature List

### 1. Real-Time Streaming

Stream intermediate output instead of waiting for the workflow to complete:

- Token-by-token LLM output in AI Studio and Vibe
- Progress updates via observable tasks
- Short-lived message persistence with replay
- NATS JetStream → SSE → Client

📖 **Docs:** [Streaming Guide](../guides/streaming) | [Streaming Consumption](../guides/streaming-consumption)

### 2. Dependency Injection

FastAPI-style `Depends()` pattern for activities:
- Sync/async functions, context managers, generators
- Single instance per worker (shared across executions)
- Automatic lifecycle management

📖 **Docs:** [Dependency Injection Guide](../guides/dependency-injection)

### 3. Distributed Rate Limiting

- Rate limits shared across **all workers** in the deployment
- Per-activity or shared via `key` parameter
- Prevents API quota exhaustion

📖 **Docs:** [Rate Limiting Guide](../guides/rate-limiting)

### 4. Concurrency Framework

`execute_activities_in_parallel()` with 3 patterns:
- **List Executor**: Known collection of items
- **Chain Executor**: Token-based pagination (S3, DynamoDB)
- **Offset Pagination**: Index-based pagination

Includes automatic continue-as-new for large datasets.

📖 **Docs:** [Concurrency Guide](../guides/concurrency)

### 5. Large Payload Handling

The platform enforces a 2MB payload limit. The SDK adds:
- **Built-in Payload Offloading**: Auto-offload to S3/GCS/Azure
- **Offloadable Fields**: Per-field offloading in activities
- **Pre-configured Encryption**: AES-GCM + key rotation out of the box

📖 **Docs:** [Handling Large Data](../guides/handling-large-data) | [Payload Encoding](../guides/payload-encoding)

### 6. Observability / Task System

- Observable `task()` context manager with state updates
- Automatic lifecycle events (started, completed, failed)
- OpenTelemetry tracing across workflows and activities
- Distributed trace queries + dashboards

📖 **Docs:** [Observability Guide](../guides/observability) | [Streaming Guide](../guides/streaming)

### 7. UI Metadata & Discovery

- `workflow_display_name`, `workflow_description` for AI Studio
- Auto-generated JSON schemas → UI input forms
- `@conversational_input` / `@input_tag` to surface workflows in Vibe

📖 **Docs:** [Workflows Guide](../guides/workflows)

### 8. Durable Agents (Mistral Plugin)

LLM agents with full durability guarantees:
- Activities as agent tools
- Multi-agent handoffs
- MCP support (stdio/SSE)
- Built-in tools: WebSearch, CodeInterpreter, ImageGeneration, DocumentLibrary

📖 **Docs:** [Durable Agents Guide](../guides/durable-agents)

### 9. HITL (Human-in-the-Loop) Interactive Workflows

Interactive workflow primitives for pausing a workflow and collecting human input:

**Core Capability:**
- **`InteractiveWorkflow` base class**: Workflows that pause and resume based on human input
- **`wait_for_input()`**: Pause workflow execution until a user provides input
- **Durability guarantee**: Workflows survive crashes while waiting for input

📖 **Docs:** [Conversational Workflows Guide](../guides/assist-workflows)

### 10. Local Execution Mode

Run workflows/activities as plain Python functions — no server required:
- Automatic retries via tenacity
- Dependency injection works locally
- **Limitations**: No durability, streaming, or human-in-the-loop

📖 **Docs:** [Local Execution Guide](../guides/local-execution)

### 11. Simplified Worker Versioning (Experimental)

- Auto-registration as the current version on startup
- No manual version management required
- Worker Controller integration for Kubernetes

### 12. Simplified Sticky Sessions

- Single decorator flag: `sticky_to_worker=True`
- No manual session management
- Works with dependency injection for state reuse

📖 **Docs:** [Activities Guide](../guides/activities#sticky-worker-sessions)

### 13. Simplified Worker Setup

- Single `run_worker()` call
- Auto-discovery of decorated activities
- Auto-registration with the control plane

📖 **Docs:** [Your First Workflow](./your-first-workflow)

### 14. Higher-Level REST API

- `/v1/workflows` — Lifecycle management, registry
- `/v1/executions` — Monitoring, control, history
- `/v1/events` — Event querying, SSE streaming
- `/v1/schedules` — Cron and one-time schedules
- `/v1/workers` — Worker capability discovery

### 15. Namespace Isolation

Multi-tenant isolation per organization and workspace:

- **Namespace Format**: `{org_id}:{workspace_id}` — each org+workspace combination gets a dedicated, isolated namespace
- **Complete isolation**: Workflows, activities, schedules, and workers are isolated per namespace
- **Automatic resolution**: SDK and API resolve the namespace from the authenticated context
- **Access control**: Namespace-level authentication and authorization
- **Independent scaling**: Each namespace can scale workers independently
- **Shared workflows**: Opt-in shared workflows callable across workspaces
- **Easy management**: Create and manage organizations and workspaces directly from AI Studio

This enables:
- **SaaS multi-tenancy**: Complete isolation between organizations and their workspaces
- **Team/project isolation**: Different workspaces within the same organization are isolated
- **Cross-workspace sharing**: Opt-in shared workflows for common utilities

### 16. AI Studio Integration

- Timeline visualization with zoom/pan
- Input forms from Pydantic schemas
- Live execution status, signal sending

### 17. Conversational Workflows and Vibe Integration

Native integration with Vibe (Mistral's chat interface):

**Workflow Invocation:**
- Workflows appear as callable assistants in Vibe
- The model can invoke workflows as tools during conversations
- Automatic parameter validation from workflow schemas

**Available Inputs:**
- **`ChatInput`**: Free-form conversational input for approvals, decisions, or feedback
- **`FormInput`**: Structured forms with typed fields and validation
  - `TextField` (with regex validation), `NumberField` (min/max constraints)
  - `DateTimeField`, `DateField`, `SingleChoice` (dropdown/select), `MultiChoice`
  - Custom UI rendering in Vibe

**Rich Outputs:**
- **`TodoList`**: Real-time progress tracking with step-by-step status updates
  - Three states: `todo`, `in_progress`, `done`
  - Context manager support for automatic status transitions
- **Canvas outputs**: Rich content rendering
  - Markdown, code with syntax highlighting, mermaid diagrams
  - SVG, HTML, React components, presentation slides
- **Streaming agent responses**: Token-by-token LLM output streamed in real-time

📖 **Docs:** [Conversational Workflows Guide](../guides/assist-workflows)
