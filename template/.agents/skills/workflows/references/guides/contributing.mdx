# Contributing to Workflows

We welcome contributions to Workflows! Here's how you can help:

## Reporting Issues

If you find a bug or have a feature request, please open an issue on our [GitHub repository](https://github.com/mistralai/dashboard/issues)

## Contributing Code

1. Open a Linear ticket in the [Workflows Team](https://linear.app/mistral-ai/team/WFL/all) with a description of the feature or bug fix you want to contribute.
2. Create a new branch for your feature or bug fix.

- Add your changes
- Add unit tests for your changes
- Add documentation for your changes

3. Make sure your code follows our coding standards and passes all tests.
4. Submit a pull request to the main repository.

## Example

If you want to add a new feature related to Workflows Scheduling, you need to make sure:

- You have a Linear ticket in the [Workflows Team](https://linear.app/mistral-ai/team/WFL/all)
- You have added unit tests for your changes in `tests/test_workflow_schedule.py`
- You have added documentation for your changes in `docs/docs/guides/scheduling.mdx`

## Share and discuss your contributions on Slack

Feel free to discuss your new features or bug fixes in #feedback-workflows.
